## Что такое W?

**W** — это open-source платформа для общения в реальном времени с самохостингом, упакованная в **один исполняемый бинарник**, написанный на Go.

Никакого Docker. Никакого Kubernetes. Никакой команды DevOps. Никакой зависимости от облачных провайдеров.  
Скачал → запустил → готово. Ваши данные остаются вашими.

W предоставляет серверы, каналы, роли, голосовые комнаты и треды — и упаковывает всё это в один портативный файл, который запускается на дешёвом VPS, Raspberry Pi или вашем ноутбуке.

```bash
# Вот и всё. Серьёзно.
./w serve
```

## Ключевые возможности

### 🏗️ Архитектура «всё в одном»

Веб-сервер, WebSocket-хаб, база данных (SQLite), файловое хранилище, API и панель администратора — всё скомпилировано в один файл. Никаких рантайм-зависимостей. Работает офлайн. Бэкап = скопировать один файл.

### 💬 Серверы и каналы

Организуйте сообщество в серверы с текстовыми каналами, каналами объявлений и тредами. Полная поддержка Markdown с подсветкой синтаксиса.

### 🔴 Всё в реальном времени

Обновления через WebSocket: сообщения, реакции, индикатор набора текста, статус онлайн — всё мгновенно. Работает за NAT, со встроенным восстановлением соединения.

### 🎙️ Голос и видео

Голосовые и видеокомнаты на базе WebRTC встроены в бинарник. Для небольших команд — без внешнего медиасервера; для больших — встроенный SFU на базе pion/webrtc.

### 👥 Роли и права доступа

Гибкая система прав: роли на уровне сервера, переопределения на уровне канала, группы администраторов. Работает как в Discord, но правила задаёте вы.

### 🤖 Bot API

Полноценный REST + WebSocket API для разработки ботов. Пишите ботов на любом языке. Альтернатива — событийные вебхуки.

### 🔍 Полнотекстовый поиск

Поиск по сообщениям во всех каналах на базе SQLite FTS5. Без Elasticsearch. Без дополнительной инфраструктуры.

### 📁 Файлы и медиа

Drag-and-drop загрузка файлов с превью изображений, аудиоплеером и встроенным воспроизведением видео. Хранилище — локальное или S3-совместимое.

### 🎨 Темы и кастомизация

Брендинг на уровне сервера: иконки, баннеры, паки эмодзи. Пользователи могут устанавливать личные темы оформления.

### 🔐 Авторизация на любой вкус

Локальные аккаунты, SSO через OAuth2 (GitHub, Google, свой IdP), magic link и двухфакторка — всё встроено.

### 📊 Встроенная панель администратора

Веб-панель по адресу `/admin`: управление пользователями, серверами, банами, хранилищем и мониторинг в реальном времени — без CLI.

### 📦 Деплой без настройки

Разумные настройки по умолчанию из коробки. Положите конфиг-файл рядом с бинарником, чтобы переопределить что угодно. Автоматический HTTPS через Let's Encrypt.

## Быстрый старт

```bash
# Скачать
curl -L https://github.com/your-org/w/releases/latest/download/w_linux_amd64 -o w
chmod +x w

# Запустить
./w serve

# Открыть в браузере
open http://localhost:8090
```

При первом запуске создаётся база данных SQLite и открывается мастер начальной настройки.

### Конфигурация (опционально)

```yaml
# w.yaml
host: "0.0.0.0"
port: 8090
data_dir: "./w_data"

tls:
  auto: true # Let's Encrypt
  domain: "chat.example.com"

storage:
  type: local # или "s3"
  path: "./uploads"

limits:
  max_upload_mb: 50
  max_servers_per_user: 10
```

## Архитектура

```
w (один бинарник)
├── HTTP-сервер       — chi router, раздаёт веб-приложение и API
├── WebSocket-хаб     — gorilla/websocket, fan-out брокер сообщений
├── SQLite (без CGo)  — modernc.org/sqlite, режим WAL, FTS5
├── WebRTC SFU        — pion/webrtc для голоса и видео
├── Файловое хранил.  — локальная ФС или S3 через minio-go
├── Auth-движок       — JWT + refresh-токены, OAuth2 клиент
├── Admin UI          — встроенная сборка SvelteKit
└── Bot runtime       — WebSocket gateway + REST API
```

W спроектирован так, чтобы его можно было **расширять как Go-библиотеку**, а не только использовать как бинарник — как PocketBase:

```go
package main

import (
    "github.com/your-org/w/core"
    "github.com/your-org/w/apis"
)

func main() {
    app := core.New()

    // Хук на создание сообщения
    app.OnMessageCreate().Add(func(e *core.MessageEvent) error {
        if e.Message.Content == "!пинг" {
            e.Channel.Send("понг!")
        }
        return nil
    })

    apis.Serve(app)
}
```

## Участие в разработке

W находится на ранней стадии и **активно принимает контрибьюторов**.

```bash
git clone https://github.com/your-org/w
cd w
go mod download
go run . serve --dev
```

Структура проекта соответствует стандартному Go-лейауту. Тесты запускаются через `go test ./...`.

Смотрите [CONTRIBUTING.md](CONTRIBUTING.md) для гайдлайнов и [good-first-issues](https://github.com/your-org/w/labels/good%20first%20issue) для точки входа.
